# Commit Message Instructions

Use the **Conventional Commit** format for all commit messages.

## Format

type(scope): description

### Rules

- Use **imperative mood**
- Good `Add authentication middleware`
- Bad `Added authentication middleware`
- Keep the **subject line under 50 characters**
- Do **not** capitalize the first letter of the description
- Do **not** end the subject line with a period
- Include a **scope** when it adds clarity (e.g., `api`, `ui`, `auth`)
- One logical change per commit

---

## Types

- **feat**: Add a new user-facing feature  
Example:
feat(auth): add JWT refresh token support

- **fix**: Fix a user-facing bug  
Example:
fix(ui): prevent crash on empty product list

- **docs**: Documentation-only changes  
Example:
docs(readme): update local setup instructions

- **style**: Code style changes (formatting, linting, no logic change)  
Example:
style(api): format controller files with prettier

- **refactor**: Code changes that neither fix bugs nor add features  
Example:
refactor(order): simplify price calculation logic

- **perf**: Performance improvements  
Example:
perf(db): optimize product query with index

- **test**: Add or refactor tests  
Example:
test(auth): add unit tests for login flow

- **chore**: Maintenance tasks (deps, configs, tooling)  
Example:
chore: update eslint and prettier configs

- **ci**: CI/CD configuration changes  
Example:
ci(github): add cache for node_modules

---

## Body (Optional)

Use the commit body for additional context when needed.

### Guidelines

- Separate subject and body with a blank line
- Use bullet points (`*`) for clarity
- Explain **why** the change was made, not just **what**

### Example

feat(api): add pagination to product list

improve performance for large datasets

add limit and offset query parameters

update response metadata

---

## Best Practices

- Write clear, concise commit messages
- Avoid combining unrelated changes in one commit
- Prefer multiple small commits over one large commit
- Commits should tell a clear story when read sequentially
